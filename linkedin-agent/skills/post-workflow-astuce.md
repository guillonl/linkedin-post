# Compétence : Post workflow/astuce d'optimisation

## Objectif
Partager un workflow complet ou une astuce qui optimise le processus de travail.

## Structure recommandée

1. **Hook workflow (1-2 lignes)**
   - Référence au schéma/visuel du workflow
   - Exemple : "Ce workflow m'a fait passer de 5h à 2h sur mes maquettes"
   - Exemple : "Voilà comment je structure mon process design + IA"

2. **Le problème initial (2 lignes)**
   - Quel processus prenait du temps
   - Pourquoi c'était inefficace

3. **Le workflow en détail (4-6 lignes)**
   - Étapes numérotées claires
   - Outils à chaque étape
   - Points de vigilance
   - Ce qui se passe entre chaque étape

4. **Bénéfices concrets (2 lignes)**
   - Gains de temps mesurables
   - Amélioration qualité
   - Ce qui change au quotidien

5. **Conseil d'adaptation (1-2 lignes)**
   - Comment adapter à d'autres contextes
   - Variantes possibles

6. **CTA pratique (1 ligne)**
   - "Quel est votre workflow pour ça ?"
   - "Des étapes que vous ajouteriez ?"

## Points d'attention
- Rester concret et applicable
- Éviter les étapes trop théoriques
- Nommer précisément les outils
- Quantifier les gains quand possible

## Ton
- Méthodique mais accessible
- "Voilà ma recette, adaptez-la"
- Pragmatique

## Longueur cible
170-240 mots
