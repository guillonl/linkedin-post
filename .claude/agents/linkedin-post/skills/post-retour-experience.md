# Compétence : Post retour d'expérience projet/apprentissage

## Objectif
Raconter une expérience concrète avec l'IA dans un projet réel.

## Structure recommandée

1. **Accroche contextuelle (1-2 lignes)**
   - Référence à l'image (résultat ou process)
   - Exemple : "Ce design que vous voyez, je l'ai prototypé en 2h avec..."

2. **Situation de départ (2-3 lignes)**
   - Projet ou défi
   - Pourquoi utiliser l'IA
   - Hésitations/questions initiales

3. **Ce que j'ai fait (3-4 lignes)**
   - Approche concrète
   - Outils utilisés
   - Process step-by-step (si pertinent)

4. **Résultat + apprentissage (2-3 lignes)**
   - Ce qui a bien marché
   - Ce qui a moins bien marché (important !)
   - Le vrai insight

5. **Conseil applicable (1-2 lignes)**
   - Ce que tu referais
   - Ce que tu changerais
   - Ce que les autres peuvent en retenir

6. **CTA échange (1 ligne)**
   - Invitation à partager leurs expériences similaires

## Éléments essentiels
- **Honnêteté** : parler aussi des ratés
- **Nuance** : l'IA aide mais ne fait pas tout
- **Pratico-pratique** : toujours un élément réutilisable

## Ton
- Storytelling authentique
- Vulnérable sur les difficultés
- Généreux sur les apprentissages

## Longueur cible
180-250 mots
